WHY? ------------->
  cmd: This directory contains the main entry point of the application. It is where the main.go file is located, which is the starting point of the worker application.
       it performs: 
        load config,
        initialize DB,
        initialize services,
        start indexing pipeline.

  internal: This directory contains the core logic and functionality of the worker application. It is organized into subdirectories based on different components or 
    features of the application, such as github, ingest, chunk, embed, storage, search, models, config, and utils. Each subdirectory contains code related to its specific
    functionality.

    1.github:clone repo,
            fetch branches,
            validate repo,
            GitHub API calls later.

    2.ingest: recursive file scanning,
            ignore rules,
            language detection,
            extracting clean files.

    3.chunk: chunking logic,
            line tracking,
            metadata generation.

    4.embed: embedding API calls,
            embedding batching,
            vector prepration
    
    5.storage: saving chunks,
            retrieving chunks,
            saving embeddings,
            repository metadata,
            retrieval quesries
    
    6.search: search logic,
            vector search,
            semantic retrieval,
            ranking

    7.models: shared data structures,
            database models

    8.config: DB URL,
            embedding API keys,
            temp directories
    
    9.utils:reusable helpers,
            file helpers,
            hash generation,
            logging utilities
  tmp: Temporary cloned repositories.


----------------------------------------------------------------Postgress SQL Database----------------------------------------------------------
  Set Up:
    1. Install PostgreSQL on your local machine or use a cloud-based PostgreSQL service.
    2. Create a new database named "askrepo".
    3. Create a new user named "askrepo" with password "askrepo" and grant all privileges on the "askrepo" database.

    SQL Commands:
    CREATE DATABASE askrepo;
    CREATE USER askrepo WITH PASSWORD 'askrepo';
    GRANT ALL PRIVILEGES ON DATABASE askrepo TO askrepo;

    --------> Local setup:
                Using Docker: 
                        docker run --name askrepo-postgres `
                        -e POSTGRES_USER=askrepo `
                        -e POSTGRES_PASSWORD=askrepo123 `
                        -e POSTGRES_DB=askrepo `
                        -p 5432:5432 `
                        -d postgres:16
                ->To check if the container is running: docker ps
                ->To stop the container: docker stop askrepo-postgres
                ->To remove the container: docker rm askrepo-postgres
                ->To start the container again: docker start askrepo-postgres
                ->After running the above command, you can connect to the PostgreSQL database using the following connection string: DATABASE_URL=postgres://askrepo:askrepo123@localhost:5432/askrepo?sslmode=disable
                -> to connect to the database using psql: docker exec -it askrepo-postgress psql -U askrepo -d askrepo
                     ==>MEANING:docker: use Docker
                                exec: run a command inside a running container
                                -it: make it interactive
                                askrepo-postgress: container name
                                psql: command to run inside container
                                -U askrepo: PostgreSQL username
                                -d askrepo: PostgreSQL database name


------------> PostgreSQL syntax:
        1. use three DB methods.
                a. Exec: for executing SQL commands that do not return rows, such as INSERT, UPDATE, DELETE, CREATE TABLE, etc.
                        example: db.Exec("INSERT INTO repositories (name, url) VALUES ($1, $2)", repo.Name, repo.URL) --> s.DB.Exec(ctx, query, args...)
                b. Query: for executing SQL commands that return multiple rows, such as SELECT.
                                example: db.Query("SELECT id, name FROM repositories") --> s.DB.Query(ctx, query, args...)
                c. QueryRow: for executing SQL commands that return a single row, such as SELECT with a WHERE clause that matches only one record.
                                example: db.QueryRow("SELECT id FROM repositories WHERE name = $1", repo.Name) --> s.DB.QueryRow(ctx, query, args...)

------------> Database Schema:
                        tables:
                                repositories: id, repo_url, name, local_path, created_at, indexed_at
                                repo_files: id, repository_id,file_name, file_path, language,content_hash, size_bytes, created_at
                                code_chunks: id, repository_id, file_id, file_path, chunk_text, start_line, end_line, embedding, created_at

---------------------------------------------------Go Syntax and Concepts---------------------------------------------------
1.Package Declaration: Every Go file starts with a package declaration. The package name is typically the same as the
        directory name. For example, if you have a file in the "github" directory, it would start with "package github".
2.Pointers: In go to pass by reference we declare arguments as pointer type using * before the type, and while calling 
        the function we pass the address of the variable using & before the variable name. This allows the function
        to modify the original variable's value.
3.Context: Context is used to manage deadlines, cancellation signals, and other request-scoped values across API boundaries
         and between processes. It is commonly used in database operations, HTTP requests, and other long-running operations
          to allow for graceful cancellation and timeout handling.
